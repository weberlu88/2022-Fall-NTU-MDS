Ref : https://medium.com/flutter-community/curse-of-dimensionality-an-intuitive-and-practical-explanation-with-examples-399af3e38e70#:~:text=Example%202%3A,dimension%20they%20can%20move%20in.

根據維基百科，我們可以知道維度詛咒是一種"現象"，它會發生在維度過高的情況下。

簡單來說，當我們的維度過高，我們就需要更多額外的資料(保持密集性，而不會過於疏鬆)去進行訓練，才能確保模型的 robustness, reliability

舉例來說:

假設我們的目的是找到一隻小狗，當在「一維空間」中我們只要一直往前走就能輕易找到小狗；當在「二維空間」中因為小狗有更多空間可以分布，散亂在X、Y座標上，因此我們需要更多的資料才比較容找到小狗。當在「三維空間」小狗學會了飛翔，它在天空中亂飛，造成資料更加疏鬆(Sparse)，如果沒有足夠的資料量我們便更難保證(Robustness、Reliability)找的到小狗

from math import log, e
def entropy(labels, base=None):
  vc = pd.Series(labels).value_counts(normalize=True, sort=False)
  base = e if base is None else base
  return -(vc * np.log(vc)/np.log(base)).sum()

Q3 (20%) 在數據科學分析架構中的決策支援階段¶

Q3-(a)什麼是模型的適應性與擴充性?¶

Ref: 教科書PPT(P35~36)¶

適應性 :¶

如字面上的意思，適應性代表模型能夠適應不同環境的能力，也就是說模型在不同的情境下，都能保持一定的預測能力。最常被討論到的就是時間、空間上的適應性。

舉例來說，模型的預測能力，是否會隨著時間下降?還是能適應這個時間的變化?這也就衍伸出了概念飄移，發現某些現象後就 re-train model，讓模型能夠適應時間。

擴充性 :¶

擴充性指的是訓練完的一個模型，是否可以將這個模型擴充到其他應用上?

舉例來說，最常聽到的應該就是 Transfer learning 了，我訓練完 A 模型(Ex: 分類100種圖片類別)之後，可以將它的參數保留下來，拿去給 B 模型(Ex: 分類貓、狗等動物)進行訓練，用來分類不同的問題。

假如以製造現場來說，就是我給 A 機台用的模型，能否擴充到 B 機台上使用

For 適應性¶

所以調整方式因加入的一評估「經濟狀態」的變量，讓我們得知當前情勢是否需要大幅的重新訓練模型。

For 擴充性¶

我覺得此數據的擴充性不錯，因為從各 Feature (如房產、職業、婚姻) 可得知一個人大致上的經濟狀況，這不只可以用來核發信用卡，也可以用來作為投資建議的依據。

我認為信用卡公司能將這個數據擴充至保險業務的推銷上，重新訓練一個保險推銷的模型應是一個不錯的應用。但是仍有一個改善空間，就是「客戶的風險傾向」，投資這一領域十分看重客戶的習性，若能新增這一特徵應能讓準確度更高。

Q4 (10%) 遺漏值填補的方法包括了統計量填補 、預測式與生成式填補¶

Q4-(a)試說明這些方法分別適用於什麼樣情形¶

統計量填補¶

  • 顧名思義，我們會用統計的方式計算出一個數值來填補 missing value，最常見的如平均值、中位數、眾數
  • 當我們想快速地填補 missing value 時，這是一個最快速的手段

預測式填補 (Ref:https://jmlr.org/papers/v18/17-073.html)¶

  • 簡單來說我們會先用現有訓練一個模型，再用此模型來預測這些 missing value。此篇論文有提到常見的方式如K -nearest neighbors, support vector machines, and decision tree 。
  • 如果我們希望填補的 missing value 與 dataset 中的其他資料有一定的關聯，可以用此方法來填補資料

生成式填補 (Ref:https://arxiv.org/abs/1806.02920)¶

  • 這是一個 GAN(Generative adversarial networks) 的應用手法，我們會訓練 GAN 的 generator 來產生資料，訓練 discriminator 來判別資料的真假；當此 generator 訓練到 discriminator 難以判別時，此生成的資料便會與真實資料十分相似，因此可以用GAN產生的資料來填補 missing data
  • 如果時間足夠，算力足夠，這應該是最有可能填補出接近真實資料的方法

若單從 Pseudo R-squ = 0.9601 來看，此羅吉斯模型的表現不錯；但是我們可以注意到 coef 的部分有強烈的負相關傾向(-326, -120, -111......)，這代表此模型其實不怎麼穩定，容易因為這些變數而造成分類錯誤。

Q5-(4)試問配適一個線性判別分析模型是否合適？若配適不佳，試說明其可能的原因為何？¶

線性判別分析模型有可能比較合適，尤其是在 muticlass 的問題中會更明顯，因為在 multiclass 的問題中羅吉斯回歸模型找尋邊界會有順序性的問題，不同順序的結果可能大不相同。

但線性判別分析模型仍可能配適不佳，主要是因為他有一個強假設「特徵近似於常態分布」，當特徵與常態分佈差異太大時，容易造成偏誤

Ref: 教科書PPT(P29)

有可能較為合適，若兩個變數分布的方向性不同時就較為合適，如圖(教科書P49)
